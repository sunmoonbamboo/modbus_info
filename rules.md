# 工作目标

从用户指定的modbus协议的pdf文件中提取关键点位信息，转换成csv格式导出

# 输入

用户指定pdf文件路径

# 输出

用户指定csv文件路径，缺省配置下存放在data/output路径下，文件名与输入文件的前缀一致

# 整体流程

用户指定输入输出路径 -> 解析输入的pdf文件 -> 提取指定的关键点位信息 -> 输出csv文件

## 解析输入的pdf文

通过minerU进行解析，使用方式参考 @minerU_demo.py，使用pipeline模式，将原始文件解析为markdown格式的文本字符串

## 提取指定的关键点位信息

需要提取的设备信息存放在 @dev_mapping.json 中，通过调用gemini-2.5-pro从上一步解析好的文本信息中提取核心信息，调用大模型时的system提示词存放在 @modbus_extract.md 中，调用大模型使用openai客户端的方式，模型相关的配置信息存放在 @.env 中

## 输出csv文件
将上一步提取出的信息输出为csv文件，存储格式参照样例文件 @data/demo.csv，如果样例文件中的某一列没有被提取出来，留空即可
