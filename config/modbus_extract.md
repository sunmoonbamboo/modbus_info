# 确定功能区和地址区间

-- 以下是Modbus点位的地址区间、功能码(modbus function)、数据类型的对应关系

地址区间 (逻辑编码),对应数据类型,英文名称,对应功能码,读写属性
00001 - 09999,线圈 (数字量输出),Coils,"01, 05, 15",可读可写
10001 - 19999,离散输入 (数字量输入),Discrete Inputs,02,只读
30001 - 39999,输入寄存器 (模拟量输入),Input Registers,04,只读
40001 - 49999,保持寄存器 (参数配置),Holding Registers,"03, 06, 16",可读可写

-- 它们的关系是固定的，因此只需要其中任何一个信息，就可以推断出其它信息；例如参考信息中能够获取到某一个点位的功能码是01，那么可以推断出它的地址区间是00001 - 09999，数据类型是线圈

# 确定数据的解析类型DataType

数据的解析类型分为WORD/BIT 两种，其判断逻辑为：

## 1、如果数据属于“线圈 (数字量输出),Coils”或者“离散输入 (数字量输入),Discrete Inputs”，换言之，它的地址区间位于“00001 - 19999”之间，则它的解析类型为WORD

## 2、如果数据是其它类型，则按照下述的规则做进一步的判断：

### 看“单位”与“范围” (最简单判据)
寄存器数值： 如果描述中出现了物理单位（如 °C、A、bar、%、min）或者一个较大的数值范围（如 0-1000）。

例子（来自你的协议）： 地址 30001，描述为“冷冻水进水温度”，单位 °C。这显然是一个具体的数值。

寄存器位解析： 通常没有物理单位，且描述的是多个互不相关的“开关”或“故障”状态。

### 看“Bit位”列是否被显式列出
这是说明书最明确的信号。

寄存器数值： 说明书的“Bit”那一列通常是空白的，或者写着“--”。

寄存器位解析： 说明书会明确标注 0, 1, 2...15。这代表这个地址像一串“糖葫芦”，一个地址里串着 16 个独立的信息。

例子： 地址 30055，左侧标注了 Bit 0, Bit 1... 这就必须按位解析。

### 看“配注说明”的逻辑
数值型： 注释会告诉你缩放比例，例如 “1000 - 100.0” 或 “Scale: 10”。这说明你需要读取整个 16 位，然后除以 10。

位解析型： 注释通常是二元对立的描述，例如 “0-无故障 1-有故障” 或 “0-关 1-开”，且针对每一个 Bit 都有不同的定义。

## 根据数据的解析类型推断其它信息，包括：

### 是否使用BIT位解析：EnableBit
在解析类型为BIT时为1，其它为0

### 是否对BIT位取反：reverseBit
-- 当EnableBit为1时：协议中说明此点位的高低位需要取反时为1，否则为0；
-- 在EnableBit为0时，此字段为空字符串

### BIT位索引：BitIndex
数据的位索引,用于确定数据在字节中的位置。
-- EnableBit为1时起效，数值范围0-16，表示从低位到高位第几个位；
-- 在EnableBit为0时，此字段为空字符串

## 根据前面的信息确定点位的地址：Address
Address为字符串类型，无需做16进制转换。

### 地址由3部分构，功能码、地址、BIT位编码，以下是2个示例，分别对应不使用解析方法为BIT和非BIT：
-- 使用BIT解析：3X0014.1
-- 不使用BIT解析：1X0001

### 功能码
Address的首字符对应Modbus的功能码：0、1、3、4区。提取时首字符代表的数字后面要加X

### 地址
中间的部分为地址的主体，在不适用BIT解析时

### BIT编码(只在使用BIT解析时有此部分)
"."后为BIT编码的索引BitIndex，在不使用BIT解析时没有这部分
