# 确定功能区和地址区间

-- 以下是Modbus点位的地址区间、功能码(modbus function)、寄存器类型的对应关系

地址区间 (逻辑编码),寄存器类型,英文名称,对应功能码,读写属性
00001 - 09999,线圈 (数字量输出),Coils,"01, 05, 15",可读可写
10001 - 19999,离散输入 (数字量输入),Discrete Inputs,02,只读
30001 - 39999,输入寄存器 (模拟量输入),Input Registers,04,只读
40001 - 49999,保持寄存器 (参数配置),Holding Registers,"03, 06, 16",可读可写

-- 它们的关系是固定的，因此只需要其中任何一个信息，就可以推断出其它信息；例如参考信息中能够获取到某一个点位的功能码是01，那么可以推断出它的地址区间是00001 - 09999，寄存器类型是线圈

# 确定数据的解析类型DataType

数据的解析类型分为WORD/BIT 两种，其判断逻辑为：

## 1、如果数据属于以下几种情况，则它的类型为BIT:
### 寄存器的类型是“线圈 (数字量输出),Coils”或者“离散输入 (数字量输入),Discrete Inputs”
### 数据的地址区间位于“00001 - 19999”之间，则它的解析类型为BIT
### 数据的功能码（分区）是：01, 05, 15，02 之一，则它的类型是BIT

## 2、如果数据不属于上述几种情况，则按照下述的规则做进一步的判断，越靠前的规则优先级越高，满足高优先级规则的情况下不需要再判断低优先级的判据：

### 看“Bit位”列是否被显式列出(最重要的标识)
这是说明书最明确的信号。

WORD解析： 说明书的“Bit”那一列通常是空白的，或者写着“--”，或者不存在Bit描述。

BIT解析解析： 说明书会明确标注 0, 1, 2...15。这代表这个地址像一串“糖葫芦”，一个地址里串着 16 个独立的信息。

例子： 地址 30055，标注了 Bit 0, Bit 1... 每个BIT位的不同点位信息，这就必须按位解析。

### 看“单位”与“范围” (最简单判据)
-- 如果描述中出现了物理单位（如 °C、A、bar、%、min）或者一个较大的数值范围（如 0-1000），解析类型为WORD
-- 如果没有物理单位，且描述的是多个互不相关的“开关”或“故障”状态，解析类型为BIT

### 看“配注说明”的逻辑
WORD型： 注释会告诉你缩放比例，例如 “1000 - 100.0” 或 “Scale: 10”。这说明你需要读取整个 16 位，然后除以 10。

BIT型： 注释通常是二元对立的描述，例如 “0-无故障 1-有故障” 或 “0-关 1-开”，且针对每一个 Bit 都有不同的定义。

## 根据数据的解析类型推断其它信息，包括：

### 是否使用BIT位解析：EnableBit
在解析类型为BIT时为1，其它为0

### 是否对BIT位取反：reverseBit
-- 当EnableBit为1时：协议中说明此点位的高低位需要取反时为1，否则为0；
-- 在EnableBit为0时，此字段为空字符串

### BIT位索引：BitIndex
数据的位索引,用于确定数据在字节中的位置。
-- EnableBit为1时起效，数值范围0-16，表示从低位到高位第几个位；
-- 在EnableBit为0时，此字段为空字符串

### 数据的转换Gain与Offset
当数据类型为WORD时，可能是使用了整数来代表浮点数，因此在有相关的数据转换或数据解析的示例时需要同步处理这2个参数

-- Gain代表解析出来的INT要乘以的系数，常见的如数据代表浮点数x10，此时Gain为0.1

-- Offset代表数字要在Gain之后做偏移

## 根据前面的信息确定点位的地址：Address
Address为字符串类型，无需做16进制转换。

### 地址由3部分构，功能码、地址、BIT位编码，以下是2个示例，分别对应不使用解析方法为BIT和非BIT：
-- 使用BIT解析：3X0014.1
-- 不使用BIT解析：1X0001

### 功能码
Address的首字符对应Modbus的功能码：0、1、3、4区。提取时首字符代表的数字后面要加X

### 地址
中间的部分为地址的主体，地址严格按照说明书中描述的数字提取，不需要处理成偏移量的形式。

### BIT编码(只在使用BIT解析，且功能码为3X或4X时有此部分)
"."后为BIT编码的索引BitIndex，只在DataType为BIT，且功能码为3X或4X时需要填写该部分，在不使用BIT解析时没有这部分
